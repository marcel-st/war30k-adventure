# WAR30K Adventure v0.6.0 — Retro Sprite System + Mission Start Fixes

This release upgrades the live gameplay renderer to a retro ALttP-style sprite-entity pipeline, improves visual readability with depth sorting, and fixes mission-start flow issues reported during playtesting.

## Highlights

- Integrated a new retro sprite-system module into the active runtime.
- Added Y-sorted character rendering for stable top-down depth overlap.
- Added daemon sine-wave hover/floating visuals.
- Added marine walk cadence with torso bob animation.
- Fixed gameplay-start lock behavior by allowing movement input to skip briefing.
- Fixed Eisenstein mission spawn issue where Garro could start in/against blocking geometry.
- Added safe spawn fallback when any configured spawn collides.

## What Changed

### Added

- `retro::SpaceMarine` sprite entity (2-frame movement cadence + bob)
- `retro::WarpDaemon` sprite entity (hover/floating effect)
- `sortEntitiesByY(...)` utility for depth ordering

### Changed

- Main gameplay render path now uses Y-sorted sprite entities.
- Runtime update flow now syncs visual entities with AI/player movement each frame.
- Documentation and gameplay images refreshed to reflect current rendering and mechanics.

### Fixed

- Briefing-to-gameplay transition now respects movement intent.
- Stage start spawn reliability improved (including Eisenstein).

## Build / Run

```bash
cmake -S . -B build
cmake --build build -j
./build/war30k_adventure
```
